Décrire l’attaque par injection SQL. L’injection SQL est l’une des vulnérabilités de sécurité des applications web les plus dangereuses et les plus courantes. Cette technique d’attaque permet d’exécuter des instructions SQL malveillantes qui contrôlent un serveur de base de données derrière une application web. Comprendre le fonctionnement de l’injection SQL est essentiel pour la création d’applications sécurisées. Définition: L’injection SQL se produit lorsqu’un attaquant insère du code SQL malveillant dans des requêtes d’application. Au lieu de traiter l’entrée utilisateur en tant que données, l’application l’exécute par erreur en tant que code, ce qui permet aux attaquants de manipuler les opérations de base de données. Fonctionnement : Les applications web construisent souvent des requêtes SQL en combinant sql statique avec une entrée fournie par l’utilisateur. Explorer la politique Azure. Azure Policy est le service de gouvernance d’Azure qui permet aux organisations d’appliquer la conformité, les normes de sécurité et les meilleures pratiques opérationnelles par le biais de stratégies en tant que code . Azure Policy évalue les ressources Azure par rapport aux règles définies, détecte la non-conformité, empêche le déploiement de ressources non conformes et corrige automatiquement la dérive de configuration. À mesure que les organisations mettent à l’échelle les déploiements Azure sur plusieurs abonnements, groupes de ressources et équipes, la maintenance de configurations de sécurité cohérentes, les contrôles de coût et les normes de conformité devient difficile. Les processus de gouvernance manuelle ne peuvent pas suivre le rythme de l’approvisionnement rapide des ressources cloud. Explorer les initiatives. Les initiatives de stratégie (également appelées ensembles de stratégies) regroupent plusieurs définitions de stratégie associées dans des groupes logiques représentant des infrastructures de conformité complètes, des bases de référence de sécurité ou des normes opérationnelles. Les initiatives simplifient la gouvernance en permettant aux organisations d’attribuer et de gérer des dizaines ou des centaines de stratégies en tant qu’unités uniques plutôt que de gérer individuellement chaque stratégie. Explorer les verrous de ressources. Les verrous de ressources protègent les ressources Azure critiques contre la suppression accidentelle ou la modification en appliquant des restrictions administratives qui remplacent les autorisations utilisateur. Même les utilisateurs disposant de rôles Propriétaire ou Contributeur ne peuvent pas supprimer ou modifier des ressources verrouillées, fournissant une protection essentielle contre les erreurs opérationnelles, les modifications non autorisées et la dérive de configuration pour l’infrastructure stratégique. Les organisations sont confrontées à des risques liés à la suppression accidentelle de ressources, notamment la suppression de base de données de production, les modifications de configuration réseau qui perturbent la connectivité et les suppressions critiques de machines virtuelles provoquant des pannes de service.
